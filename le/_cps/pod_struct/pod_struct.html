構造体、特にPODと呼ばれるC言語互換形式のものについて解説します。

複数の変数をまとめた型を作るもの

構造体の中にある変数のこと

(*).の糖衣構文

構造体基本

構造体の宣言/初期化/メンバアクセスについてです。
次のサンプルコードを実行してみましょう。
構造体基本 
#include <Windows.h>
#include <string>

struct item {
    int number;
    std::wstring name;
};

void show_item(item a) {
    MessageBoxW(
        NULL,
        a.name.c_str(),
        std::to_wstring(a.number).c_str(),
        MB_OK
    );
    return;
}

void show_item(item* a) {
    MessageBoxW(
        NULL,
        a->name.c_str(),
        std::to_wstring(a->number).c_str(),
        MB_OK
    );
    return;
}

int wWinMain(HINSTANCE, HINSTANCE, LPWSTR, int) {
    item a;
    a.number = 1;
    a.name = L"Shibuya";

    item b{ 3,L"Komabatodaimae" };

    show_item(a);
    show_item(&b);

    return 0;
}

宣言

やはり変数や関数と同様に、宣言をしなくてはなりません。
structというキーワードを使い、struct 構造体名 { 変数並び };という構文で宣言します。
構造体は複数の変数をまとめる機能が主なので、変数並びに変数宣言を記述していきます。

変数として宣言/初期化

宣言した構造体を実際に使うには、組み込み型と同様に宣言を行います。
宣言した構造体を型として、宣言された変数を構造体変数と呼んだりします。
初期化は波括弧を使い、変数並びの記述順にコンマ区切りで値を渡し、行うことが出来ます。

メンバアクセス

構造体のわかりにくいところ、構造体が持つメンバへのアクセス方法についてです。

メンバアクセス - ドット演算子

当然ですが、構造体変数が持っている変数にアクセスすることが出来ます。
構造体のメンバにアクセスする際には、変数名.メンバ名という構文を使用します。
なお、変数名.メンバ名は1つの変数を指すので、まとめて1つの変数の名前と見なせます。

メンバアクセス - ポインタ変数から

組み込み型にポインタ変数があるように、構造体にもポインタ変数が存在します。
もちろんポインタ型は構造体名*で、ポインタ変数の宣言やアドレス演算子は変わりません。
ただし、ポインタ変数からのメンバアクセスは(*変数名).メンバ名となります。
これは、ドット演算子の方が間接参照演算子よりも優先されてしまうからです。

メンバアクセス - アロー演算子

先述したように、ポインタ変数からメンバアクセスする際は(*変数名).メンバ名となります。
しかし演算子を2つ使い、丸括弧も付けないといけないのは間違いの元です。
そのため、アロー演算子という糖衣構文/シンタックスシュガーが存在します。
アロー演算子を用いると、変数名->メンバ名と記述することが出来ます。
